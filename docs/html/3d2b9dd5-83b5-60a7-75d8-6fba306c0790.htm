DesktopPet Namespace
DesktopPet is an Open Source project to get back our lovely sheep from '95.
Dedicated webpage: http://esheep.petrucci.ch.
The entire source code is available here: https://github.com/Adrianotiger/desktopPet.
This project should be flexible and easy to understand. If you want create your own pet, you need only a XML file and this file can be created directly on the webpage.
Classes
  Class Description
AboutBox
Information about application and current XML animation file
AnimationNode
All information of each single animation is stored here.
Animations
Animations class. Contains all information about the animations of the pet.
AnimationsNode
List of animations.
ChildNode
This is like a spawn but for second/child animation.
ChildsNode
List of childs.
ContextMenus
The only way to interact with the application (not the pet itself) is over the context menu.
This menu is available once you press on the tray icon (near the windows clock).
EmbeddedAssembly
This class gives some functionality if you want to embed some external DLLs. It is used to implement the NAudio.dll library.
Form2
Form2 is the main class (form) of the pet.
An ImageList will contains all images and a Timer will move the pet after a defined interval.
The animations of this form is loaded from an XML.
FormDebug
Debug form. If you start the application pressing the SHIFT-key a debug window will be started.
With this window, you can see what is happening to your pet.
FormHelp
Help Form (simple form with a Browser component, directing to eheep.petrucci.ch to display a webpage.
FormOptions
Application options. Need a redesign, so it is not documented.
HeaderNode
Main informations about the animation XML. This information is taken from the loaded xml file.
HitNode
Hit node indicate an array of next animations if the pet hit a border or has no gravity.
ImageNode
Sprite image
Install
Window form for the installer/Repair/Uninstaller. Will add shortcuts to windows.
MovingNode
Information about the moves.
NativeMethods
Native methods for the windows detection functionality. User32.dll is used for this.
NextNode
Next animation to play. Animation, Border or Gravity have 0 or more Next-nodes.
ProcessIcon
System Tray Icon. Shows an icon on the Taskbar to allow a ContextMenu.
Program
Main for the application. Once the application is started, this class will create all objects.
RootNode
This Node class is used to store the XML data using the serialize function.
SequenceNode
Sequence node.
SoundNode
To add sounds on some defined animations.
SoundsNode
List of sounds.
SpawnNode
Information about the spawn. Used to start the pet on the screen.
SpawnsNode
Node with a list of spawns.
StartUp
StartUp class. This class will initialize the entire application and define some constants.
Xml
Xml class contains all functions to read the XML file and functions to parse it.
Structures
  Structure Description
NativeMethodsRECT
Dimension structure (used for the windows size).
NativeMethodsTITLEBARINFO
Structure with the information about the title bar of the window.
StartUpTError
Error message for exceptions. It is shown in the options if an error occurs.
TAnimation
Animation structure. This contains all information about an animation.
TChild
Child structure. A second animation form can be started as child.
TMovement
Animation movement step (moves for each frame)
TNextAnimation
Information about next animation
TSequence
Each sequence contains a defined quantity of image frames. An animation is based on this sequence.
TSound
Sound structure. A sound that can be played together with the animation.
TSpawn
Spawn structure. Contains the info to start the first animation.
TValue
In the XML you can write also strings, not only numbers.
So a movement or a number can also be dynamic.
